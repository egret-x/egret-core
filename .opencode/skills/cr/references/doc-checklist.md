# 文档评审检查清单

按优先级顺序评审：A（影响最高）→ B → C。

评审员提示词会指定需要检查哪些等级。上下文加载的项目规则优先于本清单。

---

## A. 准确性

文档中存在**错误、矛盾或信息不完整**的问题。

### A1. 代码–文档一致性

- 描述的行为与实际代码实现一致
- 参数名、类型、默认值与代码一致
- 返回值与错误条件描述准确
- 描述的算法 / 处理流程与实现一致
- 版本号、格式标识、常量正确
- 取值范围与约束描述准确
- 枚举值及含义与代码定义一致
- 文件格式结构（字节偏移、字段大小等）正确

### A2. 内部一致性

- 描述同一概念的不同章节内容一致
- 文档内约束与规则无矛盾（如一处写 “必须 ≥ 0”，另一处默认值为负数）
- 多处出现的同一规则含义完全相同

### A3. 完整性

- 所有公开 API / 功能均有文档
- 新增功能或参数已在文档中体现
- 已移除或废弃功能已正确标记
- 边界情况与限制条件已文档化
- 所有条件分支均完整覆盖（无未文档化的 else 分支）
- 步骤完整，无缺失中间环节
- 未定义行为已明确标识（无文档说明的输入组合）

### A4. 引用有效性

> 不检查 URL 是否可访问。

- 内部交叉引用指向真实存在的章节或文件
- 外部链接 / URL 格式合法，无明显过时
- 引用的文件路径、工具名、命令示例真实存在

---

## B. 清晰性与结构

提升可读性、无歧义性与组织合理性的优化。

### B1. 歧义检测

- 无多种合理解读的描述
- 条件语句表述精确（should /must、may /will 区分清晰）
- 边界条件明确（包含 / 不包含、至少 / 恰好）

### B2. 简化

> 仅当同一信息在不同章节重复出现多次时才标记。

- 合并重复表述相同信息的段落或章节

### B3. 逻辑流程

- 信息按逻辑顺序呈现
- 相关主题归类在一起
- 尽量减少前向引用

### B4. 示例与图示

- 现有示例正确，与描述行为一致

### B5. 术语一致性

- 全文术语使用统一
- 术语与代码库用法一致（类名、函数名、枚举值）
- 缩写首次出现时给出定义

---

## C. 格式与风格

格式美化与风格统一。

### C1. 格式一致性

- 标题、列表、表格格式统一
- 代码片段格式正确且语法高亮合理

### C2. 语法与措辞

- 无语法错误或生硬表达
- 全文写作风格统一
- 语气符合目标读者

### C3. 章节组织

- 章节篇幅合理（不过长 / 过短）
- 目录（如有）与实际章节一致
- 废弃或过时章节已清理

---

## 排除列表

> 项目规则优先于本排除列表。

1. 不影响可读性的纯格式偏好
2. 不提升清晰性或准确性的风格改写
3. 基于假想未来需求、而非当前内容的建议
4. 超出文档既定范围的内容增补建议
5. 从示例中删除默认或详细属性 —— 规范与教程中的示例为展示可选项，刻意写得详细
