# /cr — 代码审查

对本地分支、PR、提交和文件执行自动化代码审查。

根据参数自动识别审查模式，并进入对应流程：

要么是快速的**单代理审查 + 交互式选择修复**，

要么是**多代理深度审查 + 基于风险等级的自动修复**。

所有面向用户的文本均使用用户当前使用的语言。

所有提问与选项选择**必须使用交互式对话框工具**（如 AskUserQuestion），

**不得用纯文本展示选项**。未收到用户回复前不得继续执行。

多选选项展示规则：

- ≤4 项 → 单次提问展示
- >4 项 → 按优先级或分类分组（每组 ≤4 项），在同一提示中分批展示所有组

## 路由逻辑

先执行前置检查，再**从上到下匹配第一条适用规则**：

1. `git branch --show-current` → 记录是否在 main/master 分支
2. `git status --porcelain` → 记录是否存在未提交变更
3. 检查当前环境是否支持**代理团队**（多代理并行协作、互相通信）

表格

|#|条件|动作|
|---|---|---|
|1|`$ARGUMENTS` 为 `diagnosis`|→ 读取 `references/diagnosis.md`|
|2|`$ARGUMENTS` 是 PR 编号 或 包含 `/pull/` 的 URL|→ 读取 `references/pr-review.md`|
|3|不支持代理团队|→ 读取 `references/local-review.md`|
|4|存在未提交变更|→ 读取 `references/local-review.md`|
|5|在 main/master 分支|→ 读取 `references/local-review.md`|
|6|以上都不满足|→ 执行下方提问|

每个 `→` 表示：

**读取目标文件，并以该文件作为后续唯一指令**。

忽略下方所有章节。

**不得凭记忆或习惯执行审查** ——

每个目标文件都明确规定了如何获取差异、应用修复、提交结果。

## 提问

仅提出**一个问题**：

“当前支持代理团队模式（多代理并行工作）。是否开启**审查者–验证者对抗机制 + 自动修复**的多代理审查？”

提供 4 个选项：

表格

|选项|说明|
|---|---|
|单代理 + 手动修复|单代理审查；之后交互式选择需要修复的问题|
|团队模式 + 自动修复低风险问题|多代理审查；仅自动修复最安全的问题（如空值判断、拼写、命名），其余均需确认|
|团队模式 + 自动修复低 / 中风险问题（推荐）|多代理审查；自动修复大部分问题，仅高风险问题需确认（如接口修改、架构改动）|
|团队模式 + 自动修复所有问题|多代理审查；自动修复所有问题，仅影响测试基准的问题延后处理|

### 流程转交

表格

|选项|转向|FIX_MODE|
|---|---|---|
|单代理 + 手动修复|`references/local-review.md`|—|
|团队模式 + 自动修复低风险问题|`references/teams-review.md`|low|
|团队模式 + 自动修复低 / 中风险问题（推荐）|`references/teams-review.md`|low_medium|
|团队模式 + 自动修复所有问题|`references/teams-review.md`|full|

将 `$ARGUMENTS` 传递给目标文件。

如果是团队审查（teams-review），同时传递 `FIX_MODE`（low / low_medium / full）。
