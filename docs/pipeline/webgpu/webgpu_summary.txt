一、核心类关系图
================================================================================

WebGPURenderer [主渲染器]
    ↓ 使用
WebGPURenderBuffer [渲染缓冲区] ←→ WebGPURenderContext [GPU上下文，单例]
    ├─ 包含                           ├─ 管理
    └─ WebGPURenderTarget             │  ├─ WebGPUDrawCmdManager [命令管理器]
       [离屏纹理+depth-stencil]        │  ├─ WebGPUVertexArrayObject [顶点数据]
                                      │  ├─ 着色器模块缓存 (ShaderModuleCache)
                                      │  ├─ 管线缓存 (PipelineCache)
                                      │  ├─ BindGroup缓存
                                      │  └─ GPU缓冲 (Vertex/Index/Uniform)

二、渲染管线关键数据流
================================================================================

应用场景              → Renderer.render()
                        ↓
显示对象树递归遍历    → drawDisplayObject()
                        ↓
RenderNode分派        → renderBitmap/Text/Graphics/Mesh/Group()
                        ↓
命令积累              → drawCmdManager.push*() + vao.cacheArrays()
                        ↓
统一执行              → context.$drawWebGPU()
                        ├─ 数据上传 (writeBuffer)
                        ├─ 创建CommandEncoder
                        ├─ 遍历DrawData:
                        │  ├─ ACT_BUFFER → 切换RenderPass
                        │  ├─ TEXTURE → 选管线 → drawIndexed()
                        │  ├─ RECT → 图元管线 → drawIndexed()
                        │  ├─ PUSH_MASK → stencil_push
                        │  ├─ POP_MASK → stencil_pop
                        │  └─ ...其他状态命令
                        ├─ 提交CommandBuffer
                        └─ 清理


三、关键组件职责速查表
================================================================================

┌─ WebGPURenderer (994行)
│  职责: 场景树遍历 + 各类型节点分派
│  关键方法:
│    render()                     - 主入口，push/pop缓冲栈
│    drawDisplayObject()          - 递归遍历
│    renderBitmap/Text/Mesh()     - 特定节点类型渲染
│    drawWithFilter/Clip()        - 特殊处理（mask、滤镜）
│
├─ WebGPURenderContext (1956行) [★★★ 核心]
│  职责: GPU上下文 + 完整渲染管线实现
│  关键属性:
│    device/canvasContext         - GPU核心对象
│    drawCmdManager               - 命令缓冲
│    vao                          - 顶点数据管理
│    pipelineCache                - 管线缓存
│    shaderModuleCache            - 着色器缓存
│  关键方法:
│    initWebGPU()                 - 异步初始化
│    $drawWebGPU()                - 执行所有命令（核心）
│    drawImage/drawRect/drawMesh()- 绘制接口
│    pushBuffer/popBuffer()       - 缓冲栈管理
│
├─ WebGPURenderBuffer (377行)
│  职责: 渲染缓冲区状态管理
│  关键属性:
│    globalMatrix                 - 累积变换
│    globalAlpha/globalTintColor  - 全局状态
│    rootRenderTarget             - 离屏纹理
│    stencilHandleCount           - stencil深度
│  对象池: gpuRenderBufferPool (最多6个)
│
├─ WebGPURenderTarget (274行)
│  职责: 离屏纹理+depth-stencil管理
│  关键特性:
│    惰性创建depth-stencil        - 首次pushMask时创建
│    延迟销毁                     - 避免WebGPU错误
│    尺寸同步                     - resize时重建所有纹理
│
├─ WebGPUDrawCmdManager (297行)
│  职责: 命令队列 + 批次合并
│  命令类型: TEXTURE, RECT, PUSH_MASK, POP_MASK, BLEND, ...
│  批次合并:
│    - 相同纹理 + 无滤镜 → 合并count
│    - 相同blend模式 → 覆盖冗余
│    - 连续ACT_BUFFER → 覆盖前一个
│
├─ WebGPUVertexArrayObject (317行)
│  职责: 顶点缓冲管理
│  容量: 2048 quads = 8192 vertices = 12288 indices
│  格式: 5个float32/顶点 (x, y, u, v, color+alpha)
│  支持: Quad模式 + Mesh模式两种索引
│
└─ WebGPUUtils (197行)
   职责: 工具函数
   关键: WebGPU支持检测 + Tint预乘Alpha

四、完整执行流程（精简版）
================================================================================

1. 初始化 (ApplicationBoot)
   ├─ WebGPURenderContext.getInstance()
   │  └─ initWebGPU() [async]
   │     ├─ navigator.gpu.requestAdapter()
   │     ├─ device.requestDevice()
   │     ├─ createGPUBuffers() [Vertex/Index/Uniform]
   │     ├─ createShaderModules() [编译所有WGSL]
   │     ├─ createBindGroupLayouts()
   │     ├─ createDefaultPipelines() [为每种blend创建]
   │     ├─ createFilterPipelines()
   │     └─ _initialized = true
   └─ WebGPURenderBuffer(root=true)
      └─ context.pushBuffer(this)

2. 每帧渲染循环
   MainLoop.render()
   └─ WebGPURenderer.render(displayObject, buffer, matrix)
      ├─ buffer.pushBuffer()
      ├─ drawDisplayObject(displayObject, buffer, 0, 0)
      │  └─ 递归遍历子对象
      │     ├─ 对各类RenderNode调用对应render方法
      │     ├─ renderBitmap() → context.drawImage()
      │     ├─ renderText() → Canvas→Texture→drawImage()
      │     ├─ renderGraphics() → Canvas→Texture→drawImage()
      │     └─ renderMesh() → context.drawMesh()
      │        │
      │        └─ 所有方法最终调用:
      │           ├─ drawCmdManager.push*()
      │           └─ vao.cacheArrays()
      ├─ context.$drawWebGPU() ★★★ 关键执行点
      │  ├─ 将vao.vertices上传到GPU vertexBuffer
      │  ├─ 将vao.indices上传到GPU indexBuffer
      │  ├─ 创建 commandEncoder
      │  ├─ 遍历 drawCmdManager.drawData[] 中每个命令:
      │  │  
      │  │  [ACT_BUFFER] 激活缓冲区
      │  │  ├─ 结束前一个RenderPass
      │  │  ├─ 获取目标纹理视图 (canvas或framebuffer)
      │  │  ├─ 更新投影uniform [width/2, -height/2]
      │  │  └─ beginRenderPass()
      │  │
      │  │  [TEXTURE] 纹理绘制
      │  │  ├─ 选管线: texture_[blendMode] 或 filter_[type]
      │  │  ├─ 创建BindGroup (uniform + sampler + texture)
      │  │  ├─ (如有滤镜) 创建filter uniform BindGroup
      │  │  ├─ setStencilReference(if needed)
      │  │  └─ drawIndexed(count * 3)
      │  │
      │  │  [RECT] 矩形/图元
      │  │  ├─ 选管线: primitive_[blendMode]
      │  │  ├─ 使用cached primitive BindGroup
      │  │  └─ drawIndexed(count * 3)
      │  │
      │  │  [PUSH_MASK] Stencil增
      │  │  ├─ (首次)重启RenderPass + 附加depthStencil
      │  │  ├─ 使用 stencil_push 管线 (incr-clamp)
      │  │  ├─ colorMask = 0 (不写颜色)
      │  │  └─ drawIndexed()
      │  │
      │  │  [POP_MASK] Stencil减
      │  │  ├─ 使用 stencil_pop 管线 (decr-clamp)
      │  │  ├─ (最后一个)重启RenderPass + 移除depthStencil
      │  │  └─ drawIndexed()
      │  │
      │  │  [BLEND] 混合模式
      │  │  └─ currentBlendMode = value
      │  │
      │  │  [其他] CLEAR, SCISSOR等状态
      │  │  └─ 对应的状态管理
      │  │
      │  ├─ renderPassEncoder.end()
      │  ├─ device.queue.submit([commandEncoder.finish()])
      │  ├─ drawCmdManager.clear()
      │  └─ vao.clear()
      │
      ├─ buffer.popBuffer()
      └─ (nestLevel===0时) 清理对象池

五、关键数据结构
================================================================================

WebGPU GPU缓冲:
  vertexGPUBuffer    32KB  (2048 quads * 4 verts * 20bytes)
  indexGPUBuffer     24KB  (2048 quads * 6 indices * 2bytes)
  uniformBuffer     256B   (projectionVector: vec2<f32> + padding)
  filterUniformBuffer512B   (ColorTransform/Blur/Glow的uniform)

管线命名规则:
  [type]_[blend]_[stencil_optional]
  ├─ type: texture, primitive, filter_colorTransform_frag, stencil_push, ...
  ├─ blend: source-over, lighter, destination-out, ...
  └─ stencil: [可选] _stencil 后缀

着色器种类:
  ✓ default_vert (所有管线共用顶点着色器)
  ├─ 顶点坐标变换: pixel → [-1,1]
  └─ 使用projectionVector = [width/2, -height/2]
  
  ✓ texture_frag (纹理采样，UV翻转)
  ✓ primitive_frag (纯色矩形)
  ✓ colorTransform_frag (颜色矩阵滤镜)
  ✓ blur_frag (可分离的高斯模糊)
  ✓ glow_frag (发光效果)
  ✓ texture_etc_alphamask_frag (ETC纹理透明度处理)

混合模式映射:
  source-over    → src=one, dst=one-minus-src-alpha
  lighter        → src=one, dst=one  (相加)
  destination-out → src=zero, dst=one-minus-src-alpha  (反向减)
  destination-in → src=zero, dst=src-alpha
  lighter-in     → src=src-alpha, dst=one-minus-src-alpha

六、性能优化手段
================================================================================

✓ 批次合并
  ├─ 相同纹理自动合并 (除非有滤镜)
  ├─ 冗余blend命令剔除
  └─ 结果: UI界面通常 100+ draw calls → 10-50

✓ 缓存策略
  ├─ 管线缓存: 预编译 → 永久存储
  ├─ BindGroup缓存: Primitive BindGroup复用，Texture BindGroup按需创建
  ├─ TextureView缓存: 每个GPUTexture一个View
  └─ Sampler缓存: 仅2个 (linear/nearest)

✓ 数据管理
  ├─ 批量上传: 一次writeBuffer()上传所有数据
  ├─ 字节对齐: 4字节对齐后上传
  ├─ 延迟销毁: 纹理不立即destroy，下一帧批量销毁
  └─ 对象池: WebGPURenderBuffer最多保持6个

✓ GPU内存优化
  ├─ Stencil惰性启用: 首次pushMask时才创建depth-stencil
  ├─ Uniform复用: 投影uniform仅当参数变化时更新
  └─ 默认空纹理: 备用16x16白色纹理

七、高级特性
================================================================================

★ Stencil遮罩 (支持嵌套)
  pushMask(x, y, w, h)
    → drawCmdManager.pushPushMask()
    → stencilHandleCount++
  
  在$drawWebGPU()中:
    ├─ 首次: 重启RenderPass + 附加depthStencil
    ├─ 使用 stencil_push 管线
    ├─ stencilFront: compare='equal', passOp='increment-clamp'
    └─ drawIndexed()
  
  popMask()
    → 使用 stencil_pop 管线 (decrement-clamp)
    → stencilHandleCount--
    → (最后) 重启RenderPass移除depthStencil
  
  后续绘制:
    setStencilReference(stencilHandleCount)
    → 仅stencil值==参考值的像素通过

★ Scissor测试 (优化scrollRect)
  enableScissor(x, y, w, h)
    → drawCmdManager.pushEnableScissor()
    → renderPassEncoder.setScissorRect()
  
  disableScissor()
    → setScissorRect(0, 0, width, height)

★ 滤镜管线
  支持: ColorTransform, Blur(可分离), Glow
  处理方式:
    ├─ 单一colorTransform: 直接标记，在TEXTURE指令中处理
    ├─ 其他情况: 创建offscreen buffer → 链式处理
    └─ 多个滤镜: 逐个应用，中间buffer→最终buffer

八、纹理生命周期
================================================================================

创建: getGPUTexture(bitmapData)
  ├─ 检查 bitmapData["gpuTexture"]
  └─ 不存在则 createTexture()
     ├─ device.createTexture()
     ├─ copyExternalImageToTexture() [异步]
     └─ 标记 UNPACK_PREMULTIPLY_ALPHA_WEBGL=true

更新: updateTexture(texture, bitmapData)
  ├─ 检查尺寸
  ├─ 尺寸变化: deleteGPUTexture() → createTexture()
  └─ 否则: copyExternalImageToTexture()

销毁: deleteGPUTexture(texture)
  ├─ 移除textureView缓存
  └─ 加入销毁队列 (延迟销毁)
     → flushDestroyTextures() [下一帧开始时执行]

九、常见问题和解决方案
================================================================================

Q: 纹理倒立/显示错误
A: WebGPU纹理原点在左上 vs WebGL在左下
   → 着色器中 uv.y = 1.0 - uv.y (已实现)
   → VAO中vStart/vEnd计算已翻转 (已实现)

Q: depthStencilTexture尺寸不匹配
A: resize()时需要重建depthStencil
   → rootRenderTarget.resize() 处理
   → ACT_BUFFER时 ensureDepthStencilSize()

Q: 批次合并不生效
A: 检查:
   ├─ 纹理之间是否有filter (有则不合并)
   ├─ blend模式是否一致
   └─ 纹理对象引用是否相同

Q: Stencil mask 性能差
A: 避免超过8层嵌套，必要时用offscreen替代

Q: 异步初始化导致黑屏
A: await context.ensureInitialized() 或延迟draw调用

十、扩展方式
================================================================================

添加新着色器:
  1. WGShaderLib.ts 中定义shader代码
  2. createShaderModules() 中编译
  3. createFilterPipelines() 中创建管线
  4. getFilterPipeline() 中分派

添加新混合模式:
  1. initBlendMode() 中注册GPUBlendState
  2. createDefaultPipelines() 中为每种blend创建管线变体
  3. setGlobalCompositeOperation() 使用

自定义滤镜:
  继承 egret.Filter
  ├─ 设置 type="custom"
  ├─ 定义 $uniforms
  └─ 在 buildFilterUniformData() 处理

十一、与WebGL的主要区别
================================================================================

特性             WebGL                    WebGPU
─────────────────────────────────────────────────────
管线             动态GLSL编译              预编译WGSL
命令             立即执行                  批量提交
混合模式         glBlendFunc              Pipeline.fragment.blend
Stencil          glStencilFunc/Op         depthStencilAttachment
Sampler          内联到纹理                独立GPUSampler对象
Buffer绑定       glBindBuffer(target)     setVertexBuffer/setIndexBuffer
数据上传         bufferData/SubData       device.queue.writeBuffer
Viewport         gl.viewport              RenderPass + setScissorRect
FrameBuffer      WebGLFramebuffer         纹理 + RenderPass attachment
异步             无                       设备创建/初始化异步
错误             立即异常                 延迟或静默

总结: WebGPU更现代、更显式、批量化程度更高，性能潜力更大
